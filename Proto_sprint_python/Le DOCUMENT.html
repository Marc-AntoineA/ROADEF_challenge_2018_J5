class Stack:    
    def __init__(self, L):
        self.list = L
        self.n = len(L)
        self.begin = 0 # premier lment
        self.end = self.n  # juste aprs le dernier
        
    def pop(self):
        """
            Renvoie l'lment en tte et avance l'item
        """
        if not self.is_empty():
            self.begin += 1
            return self.list[self.begin - 1]
        
    def is_empty(self):   
        return self.end - self.begin == 0
    
    def push(self):
        """
            Ajoute artificiellement l'lment en tte 
            (dcale juste le pointeur begin)
        """
        if self.begin > 0:
            self.begin -= 1
        
    def reset(self, begin = 0, end = -1):
        self.begin = begin
        
        if end == -1:
            self.end = self.n
        else:
            self.end = end
            
    def top(self):
        if not self.is_empty():
            return self.list[self.begin]
        else:
            return -1

def random_seq(stacks):
    seq = []
    
    for k in range(len(stacks)):
        stacks[k].reset()
    n = len(stacks)
    k = 0
    while k < sum([s.n for s in stacks]):
        p = randint(0, n - 1)
        if not stacks[p].is_empty():
            k += 1
            seq.append(stacks[p].pop())
            
    for k in range(n):
        stacks[k].reset()
    return seq

Création de l'arbre à partir d'un cutting pattern¶

Algorithme plutt simple, il construit un arbre prioritaire dans le sens o chaque coupe est faite ds que possible.

Une coupe est considre comme valide si elle vrifie plusieurs conditions :

  • Coupes guillotines : la coupe ne traverse aucune pice, ni aucun dfaut
  • Ordre admissiple : la coupe ne spare pas des squences de pices (1, 2, 3, 4, 5) ne peut tre dcoupe en (1, 2, 4) et (3, 5).

    Remarque : Cette contrainte porte sur la squence donne en paramtre, mais une autre squence pourrait tre tre admissible.

  • 1-cut :
    • la largeur minimale entre deux 1-coupe conscutives doit tre de minXX (= 100), sauf pour les dchets
    • la largeur maximale entre deux 1-coup conscutes doit tre de maxXX (= 3500), sauf pour les dchets (ou la zone rsiduelle)
  • 2-cut : la hauteur minimale entre deux 2-coupes est de minYY (= 100), sauf pour les dchets
  • Les zones de dchets sont de dimensions au minimum $20 \times 20$

Attention, la coupe guillotine sur un dfaut ainsi que la condition $20\times20$ ne sont pas encore pris en compte.

Emplacements¶

L'objectif des fonctions ci-dessous est de russir obtenir, pour un item et des items dj placs quels sont emplacements possibles (et lesquels sont possibles pour chaque orientation).

Un emplacement vrifie trois conditions :

  • La condition de l'toile verte, cns pour vrifier la contrainte d'ordre (et tre un emplacement dcent)
  • La condition du monstre rouge, cns pour viter les dfauts (et tre un emplacement dcent)
  • La condition de dimension, qui garantit que la pice tient.

Une quatrime condition, relative l'existence d'un arbre admissible et aux contraintes 1-cut, 2-cut et de profondeur maximale, est gre par la fonction gen_tree.

def cost_roadef(items_cut):
    s = 0
    n = len([i for i in range(len(items_cut)) if len(items_cut[i]) > 0])
    for bin_id in range(n):
        for it in items_cut[bin_id]:
            s += width_item(int(it[0]), True, batch_mat)*height_item(int(it[0]), True, batch_mat)
        
    list_residuals = [item[1] + width_item(item[0], item[3], batch_mat) for item in items_cut[n - 1]]
    # Normalement, a ne devrait pas arriver
    if len(list_residuals) == 0:
        return 0
    
    return ((n - 1)*widthPlates + max(list_residuals))*heightPlates - s

Résolution exhaustive¶

La fonction glouton alatoire n'optimise que sur un seul triplet (pice, emplacement, rotation) et fournit dj de bons rsultats. Toutefois, des trous subsistent, dans lesquels il serait intressant de mettre des pices.

Afin de limiter ces trous, l'ide serait de raliser une recherche exhaustive sur l'arbre des solutions, constitu des triplets (pice, emplacement, rotation). Malheureusement, cela semble impossible combinatoirement, notamment parce que je ne pense pas disposer (surtout maintenant) facilement de bonnes bornes afin de raliser un algorithme de branch and bound.

Par contre,je ne pense pas qu'il soit ncessaire de descendre trop profondemment dans l'arbre : une profondeur de 2 ou 3 devait tre suffisante pour d'une part obtenir de bons rsultats, et d'autre part conserver une complexit rduite (la complexit demeure polynomiale, de degr la profondeur tudie).

Algorithmiquement, il s'agit de construire une fonction rcursive pour raliser cette recherche exhaustive, pour une profondeur donne.

L'algorithme en pseudo-code, actuellement utilis dans la fonction glouton alatoire (qui porte trs mal son nom).

Pour chaque stack disponible
    Calculer les emplacements disponibles
        Pour chaque rotation 
            Essayer de construire un arbre
            Si c'est possible, calculer le score
            Sinon, mettre le score  inf
Renvoyer le meilleur triplet (pice, emplacement, rotation)

L'algorithme, en pseudo code, pour faire une recherche exhaustive, sans optimisation

function score_recursif(stacks, stacks_available, hull, items_cuts, profondeur):
    Initialiser meilleur_score  inf
    Initialiser meilleur_triplet

    Si des triplets sont disponibles ou si la profondeur maximale n'est pas atteinte
        Pour chaque triplet (pice, emplacement, rotation) possible    
            Rajouter  items cut le triplet courant
            Mettre  jour stacks_available
            Mettre  jour hull
            Calculer le score de score_recursif(stacks, stacs_available, hull, items_cuts, profondeur)
            Si ce score est infrieur  meilleur score
                Mettre  jour meilleur_score et meilleur_triplet                
            Rinitialiser hull, stacks_available, items_cut

       Rajouter  items_cut le meilleur triplet et mettre  jour hull et stacks_available
       Renvoyer les meilleures solutions actuelles (triplet et score)
    Sinon
        Calculer et renvoyer le score de la solution courante ainsi que le triplet vide
